# Services Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:executing-plans (inline). Steps use checkbox (`- [ ]`).

**Goal:** `/services` overview + a data-driven detail template for the 9 service pages with Service + BreadcrumbList + FAQPage JSON-LD.

**Architecture:** Expand `data/services.ts`; add JSON-LD helpers + a `Breadcrumb` primitive; build `/services` and `/services/[slug]` (prerendered via `generateStaticParams`). Reuse primitives, `Reveal`, hue `Eyebrow`.

**Tech Stack:** Next.js 16 App Router, TS, Tailwind v4.

## Global Constraints

- Copy: defensible, in-brand, qualitative — no hard market stats. NAP/founder from `siteConfig`.
- Spectrum accent-only; reduced-motion safe; keyboard + visible focus; editorial hierarchy (open-design craft).
- All routes prerender; per-page unique metadata.
- Verify per task: `type-check` + `lint` + `build` clean; final confirmed on `/services` + a sample detail. Commit per task (branch `feat/services`).

## Tasks

### Task 1: Expand `data/services.ts`
- [ ] Replace `data/services.ts`: keep slugs/names/promises/hues; add `overview`, `context`, `keyServices`, `ecosystem {label,href,note}`, `project {name,href}`, `faqs[]` for all 9. Ecosystem hrefs target `/ecosystem/<slug>`; project hrefs target `/projects/<slug>` from `data/projects.ts` where sensible. Export `Service` type and a `getService(slug)` helper.
- [ ] `npm run type-check`. Commit.

### Task 2: JSON-LD helpers + Breadcrumb primitive
- [ ] `lib/json-ld.ts`: add
  - `serviceLd(s: Service)` → `{ "@context", "@type":"Service", name, description, serviceType, areaServed:"Bangladesh", provider: organizationLd() }`.
  - `breadcrumbLd(items: {name,url}[])` → BreadcrumbList with `ListItem` positions.
  - `faqLd(faqs: {q,a}[])` → FAQPage with `Question`/`Answer`.
- [ ] `components/primitives/breadcrumb.tsx`: `Breadcrumb({ items: {label,href?}[], className? })` — `nav[aria-label="Breadcrumb"]`, mono, hairline `/` separators, last item `aria-current="page"`, links via `next/link`. Export from `components/primitives/index.ts`.
- [ ] type-check + lint. Commit.

### Task 3: `/services` overview
- [ ] `app/services/page.tsx`: `metadata` (title "Services", description, canonical `/services`, OG). Hero (Eyebrow accent="spine" "The spectrum", `t-h1` title, intro). Editorial list of `services` — each a `Link` row: `λ NN` (hue) + name (`t-h3`) + promise; `Reveal` per row. Closing CTA (Book a consultation). Reuse `Section`/`Container`.
- [ ] type-check + lint + build. Commit.

### Task 4: `/services/[slug]` detail template
- [ ] `app/services/[slug]/page.tsx`:
  - `export function generateStaticParams()` → `services.map(s => ({ slug: s.slug }))`.
  - `export async function generateMetadata({ params })` → look up service; if missing return base; else title = name, description = overview, canonical `/services/<slug>`, OG/Twitter.
  - Component: `const s = getService(slug); if (!s) notFound();` Render: `Breadcrumb` (Home/Services/name) + hero (hue Eyebrow `λ NN`, name `t-display`/controlled clamp, promise) → context → key services list → ecosystem connection (Link) → project proof card (Link) → FAQ (2–3) → CTA → prev/next nav (wrap around `services`). Inject `<script type="application/ld+json">` with `[serviceLd(s), breadcrumbLd(...), faqLd(s.faqs)]`.
  - `params` is a Promise in Next 16 — `const { slug } = await params;`.
- [ ] type-check + lint + build (confirm 9 `/services/<slug>` prerender). Commit.

### Task 5: Verify
- [ ] `npm run dev`; confirm `/services` lists 9 → each links to a detail; a sample detail (`/services/funding-investment-advisory`) renders all sections, JSON-LD (Service/Breadcrumb/FAQPage) present, prev/next works; dev log clean. Commit any fixes.
